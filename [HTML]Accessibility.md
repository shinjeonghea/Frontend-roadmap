## 접근성

- 웹 사이트에서 제공하는 **정보를 차별 및 제한 없이 동등하게 이용할 수 있도록 보장하는 것**

  - 장애인 및 고령자 등을 포함한 모든 사람
  - 다양한 플랫폼 및 장치, 웹 브라우저 등의 모든 환경
- 기본적으로 **HTML을 올바르게 사용하면 접근성이 향상** 된다.

<BR>

:pushpin: **즉, 접근성을 위해서 올바른 HTML을 사용하는 것이 중요하다. 이에 대한 설명을 하기 위해 [Sematic HTML](./%5B%5DSemantic%20HTML.md) 에 대한 설명을 알아보자**

<br>

## 웹 접근성 준수 시 기대 효과

1. Sematic HTML의 사용으로 SEO도 개선
2. 장애인, 고령자 등을 포함한 사용자층 확대
3. 규정과 법적 요구 사항에 대한 준수
4. 다양한 환경, 새로운 기기에서의 이용
5. 개발 및 운용의 효율성 향상
6. 사회 공헌 및 복지 기업으로서의 기업 이미지 향상

<BR>

## 웹 컨텐츠 접근성 지침

웹 접근성 향상을 위해 노력하는, W3C의 산하 단체 WAI(Web Accessibility Initiative)의 웹 콘텐츠 접근성 지침<BR><sub>WAI : 시각/청각 기능 등에 장애를 지닌 사람들이 일반인과 동등하게 웹에 접근하도록 노력하는 단체 </sub><br>

- #### 인지성

  정보와 사용자 인터페이스 요소는 시각/청각 기능 등의 장애를 지닌 사람들이 인지할 수 있도록 사용자에게 표시될 수 있어야한다.
  
  - 모든 텍스트가 아닌 콘텐츠의 대체 텍스트를 인쇄, 점자, 음성, 기호 도는 간단 언어와 같은 형태로 제공할 수 있어야 한다.
  - 시간을 바탕으로 한 미디어에 대한 대안을 제공할 수 있어야한다.
  - 정보와 구조의 손실 없이 콘텐츠를 다른 방식으로 표현할 수 있어야한다.
  
- #### 운용성

  사용자 인터페이스 요소와 탐색은 운용 가능해야 한다.

  - 키보드로 모든 기능을 사용할 수 있도록 해야 한다.
  - 읽기 및 콘텐츠를 사용하는 사용자에게 충분한 시간을 제공해야 한다.
  - 사용자가 탐색하고, 콘텐츠를 찾는 것을 도와주는 방법을 제공해야 한다.

- #### 이해성

  정보와 사용자 인터페이스 운용을 이해할 수 있어야 한다.

  - 텍스트 콘텐츠를 판독하고 이해할 수 있도록 만들어야 한다.
  - 웹 페이지의 탑재와 활용을 예측 가능한 방법으로 제작해야 한다.
  - 사용자의 실수를 방지하고 수정할 수 있도록 도와야 한다.

- #### 견고성

  콘텐츠는 사용자의 다양한 요구에도 해석될 수 있도록 충분한 내구성을 가져야한다.

  - 보조 기술을 포함한 현재 및 미래의 사용자 요구에 대한 호환성을 극대화 한다.

## 웹 접근성을 고려한 HTML 권장사항

**1. 크로스 브라우징이 가능하도록한다.**

- 크로스 브라우징이란, 웹 표준 기술을 적용하여 웹페이지가 서로 다른 운영체제나 플랫폼에서 이상없이 구현되도록 하는 기술

**2. 모든 값은 따옴표('')로 묶어준다.**

- 속성값을 구분할 경우 따옴표를 누락하면 정보가 무시될 수 있다.

**3. 단일 태그의 마무리는 항상 '/>'로 해준다.**

~~~html
<br/>
~~~

**4. 캐릭터셋을 반드시 지정해준다.**

- 인코딩을 정확하게 알려주기 위해서 이다.

  ~~~html
  <meta charset="utf-8" >
  ~~~

**5. 이미지 태그에는 alt를 반드시 지정한다.**

~~~html
<img src="thumb.jpg" alt="즐거운 여름 휴가" />
~~~

**6. 색상은 RGB코드로 지정한다.**

**7. HTML 코드 들여쓰기를 한다.**

~~~~HTML
<div>
	<p>
		<strong> 
            안녕
        </strong>
    </p>
</div>
~~~~

**8. 이미지 파일명은 의도가 드러나도록 작성한다.**

**9. class의 이름을 지을 때, 상위 태그는 box를 포함하여 지정한다.**

- 가장 큰 단위의 요소인 div 태그의 클래스 명은 wrap으로 통일하는 것이 좋다.

  ~~~html
  <div class=“wrap”>
      전체 내용
  </div>
  ~~~

- 가장 큰 단위 아래에 모듈별 단위는 box를 포함하는 것이 좋다.

  ~~~html
  <div class=“wrap”>
      <div class=“box-1”>
          1번 내용
      </div>
      <div class=“box-2”>
          2번 내용
      </div>
  </div>
  ~~~

**10. 제목(h) 태그를 적절하게 사용한다.**

**11. 사이즈는 px(픽셀) 단위를 주로 사용한다.**

~~~css
div{
    margin: 10px 0 10px 0;
}
~~~

 **12. 표 형식을 나타낼 때에는 가급적 table 태그를 사용한다.**
